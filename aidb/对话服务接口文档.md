# 对话服务 `/chat` 接口文档

> 核心问答接口详解，包含每个接口的请求参数、响应格式、执行逻辑

---

## 接口总览

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| **核心问答** | POST | `/chat/get_answer` | SSE流式问答入口 |
| **停止聊天** | POST | `/chat/stop_chat` | 停止运行中的任务 |

## 1. POST `/chat/get_answer` - 核心问答接口

### 📌 基本信息

| 属性 | 值 |
|------|-----|
| **路径** | `/chat/get_answer` |
| **方法** | POST |
| **认证** | JWT Token (Bearer) |
| **响应格式** | `text/event-stream` (SSE流式) |
| **内容类型** | `text/event-stream` |

---

### 📌 请求参数

**请求体 (JSON)**:

```json
{
  "query": "string",           // ✅ 必填 - 用户问题
  "chat_id": "string",        // ✅ 必填 - 对话ID（用于区分不同对话）
  "uuid": "string",           // ✅ 必填 - 唯一标识一次问答
  "qa_type": "string",        // ✅ 必填 - 问答类型
  "file_list": [],            // ❌ 可选 - 附件列表
  "datasource_id": 1           // ❌ 可选 - 数据源ID（DATABASE_QA时必填）
}
```

**参数详细说明**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `query` | string | ✅ | 用户的自然语言问题 |
| `chat_id` | string | ✅ | 对话会话ID，支持多轮对话 |
| `uuid` | string | ✅ | 唯一标识这次问答的UUID |
| `qa_type` | string | ✅ | 问答类型，决定调用哪个Agent |
| `file_list` | array | ❌ | 上传的附件列表（用于ExcelAgent和DeepAgent） |
| `datasource_id` | integer | ❌ | 数据源ID（DATABASE_QA/REPORT_QA时需要） |

---

### 📌 qa_type 问答类型

| qa_type | Agent | 说明 | 必需参数 |
|----------|-------|------|----------|
| `COMMON_QA` | CommonReactAgent | 通用知识问答 | - |
| `DATABASE_QA` | Text2SqlAgent | 数据库问答 | datasource_id |
| `FILEDATA_QA` | ExcelAgent | Excel文件分析 | file_list |
| `REPORT_QA` | DeepAgent | 深度研究报告 | datasource_id |

---

### 📌 响应格式 (SSE)

**响应内容类型**: `text/event-stream`

每个SSE消息格式:
```
data:{JSON_OBJECT}\n\n
```

**消息类型 (dataType)**:

| dataType | 说明 | 示例 |
|----------|------|------|
| `t02` | 文本回答 | 用户的自然语言回答 |
| `t03` | 溯源信息 | SQL来源、数据来源等 |
| `t04` | 业务数据 | 表格数据、图表配置等 |
| `t11` | 任务ID | 方便后续点赞等操作 |
| `t14` | 步骤进度 | 各步骤的执行状态 |
| `t99` | 流结束标记 | 表示整个流已结束 |

**messageType 消息子类型**:

| messageType | 说明 |
|--------------|------|
| `begin` | 开始输出 |
| `continue` | 继续输出 |
| `end` | 结束输出 |

---

### 📌 响应示例

```javascript
// 步骤开始
data:{"data":{"messageType":"begin","content":"正在分析您的需求..."},"dataType":"t02"}

// 步骤进行中
data:{"data":{"messageType":"continue","content":"正在连接数据库..."},"dataType":"t02"}

// 步骤进度
data:{"data":{"messageType":"continue","content":"表结构检索..."},"dataType":"t14"}

// 业务数据（图表配置等）
data:{"data":{"messageType":"continue","content":"{...}"},"dataType":"t04"}

// 流式结束
data:{"data":"DONE","dataType":"t99"}
```

## 3. POST `/chat/stop_chat` - 停止聊天

### 📌 基本信息

| 属性 | 值 |
|------|-----|
| **路径** | `/chat/stop_chat` |
| **方法** | POST |
| **认证** | JWT Token (Bearer) |
| **响应格式** | JSON |

---

### 📌 请求参数

**请求体 (JSON)**:

```json
{
  "task_id": "string",    // 任务ID（通常为用户ID）
  "qa_type": "string"     // 问答类型
}
```

---

### 📌 响应格式

```json
{
  "success": true,
  "message": "任务已停止"
}
```

**响应说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| `success` | boolean | 是否成功停止 |
| `message` | string | 状态消息 |

---

### 📌 执行逻辑

```
┌─────────────────────────────────────────────────────────────────┐
│                   stop_chat() 函数                               │
└─────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼

│  Step 1: 验证 Token                                              │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │  从请求头获取 Authorization Bearer Token                   │ │

│  └───────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────┐
│  Step 2: 路由分发                                                │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │  根据 qa_type 调用对应 Agent 的 cancel_task()            │ │
│  │                                                           │ │
│  │  COMMON_QA    → common_agent.cancel_task(task_id)        │ │
│  │  DATABASE_QA  → sql_agent.cancel_task(task_id)           │ │
│  │  FILEDATA_QA  → excel_agent.cancel_task(task_id)        │ │
│  │  REPORT_QA    → deep_agent.cancel_task(task_id)         │ │
│  └───────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────┐
│  Step 3: 取消任务                                                │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │  running_tasks[task_id]["cancelled"] = True             │ │
│  │  → Agent 在下次迭代检查该标志并退出                        │ │
│  └───────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────┘
```

## 4. 错误处理

### 4.1 HTTP 错误响应

| 状态码 | 说明 |
|--------|------|
| 401 | 未授权 / Token 无效 |
| 403 | 无数据源权限 |
| 500 | 系统异常 |

**示例** (无权限):
```json
{
  "code": 403,
  "msg": "您没有访问该数据源的权限，请联系管理员授权。",
  "data": null
}
```

data:{"data":{"messageType":"continue","content":"发生错误: ..."},"dataType":"t02"}

### 5.2 JavaScript fetch 示例

```javascript
// SSE 流式请求
const response = await fetch('http://localhost:2048/sanic/chat/get_answer', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer <your_jwt_token>',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    query: '去年销售额是多少？',
    chat_id: 'chat_001',
    uuid: crypto.randomUUID(),
    qa_type: 'DATABASE_QA',
    datasource_id: 1
  })
});

// 读取 SSE 流
const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  
  const chunk = decoder.decode(value);
  const lines = chunk.split('\n');
  
  for (const line of lines) {
    if (line.startsWith('data:')) {
      const data = JSON.parse(line.slice(5));
      console.log(data);
    }
  }
}
```
